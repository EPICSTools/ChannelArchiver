Channel Archiver

The Channel Archiver is an archiving system for EPICS. It saves the information available via Channel Access, i.e. the time stamp, status and value of individual channels, together with control information (limits, units, ...). This includes all data types available via Channel Access: float, double, int, string, enumerated types, scalars as well as arrays.


Engine
Archiver engine, collects data via Channel Access and stores it in disk files.
There are several options for data retrieval. Since all tools use the archiver I/O library, they currently need access to the data files, be it on a local file system or via NFS or Samba.
WinBrowser
WIN32 archive browser, allows interactive browing or archived data as well as spreadsheet export.
xarr, Striptool
These tools, developed by Chris Larrieu (JLab), are available with plugins which use this Archiver's LibIO.
CGIExport
CGI plugin for web server, allows spreadsheet-download and simple online plotting (via gnuplot)
casi
Based on SWIG, the Channel Archiver Scripting Interface "casi" provides scripting languages (tcl, python, ...) with access to the archive data. Includes Python-based Cardminer, an archive file management tool.
Manager
Command-line tool to test the I/O Lib and manage archive files, to be replaced by cardminer.
LibIO
I/O library to access CA files: Basic I/O tools for proprietary archive data file format as well as generic classes based upon that like SpreadSheetExporter
../Tools
C++ classes shared by CA and some other code
